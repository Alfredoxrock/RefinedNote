Standard install scenarios

Provide EXE return code values for the scenarios mentioned below.

Scenario EXE return code value Notes / Action
Installation cancelled by user — The install operation was cancelled by the user. 1 Action: No action required. User cancelled the install.
Application already exists — The application already exists on the device. 2 Action: Inform user or run as upgrade. Check existing installation state.
Installation already in progress — Another installation is already in progress. User needs to complete the installation before proceeding with this install. 3 Action: Ask user to wait until the other installer finishes or reboot and retry.
Disk space is full — Not enough disk space to complete installation. 4 Action: Free disk space and retry installation. Provide link to disk cleanup instructions.
Reboot required — A restart is required to complete the install. 5 Action: Prompt user to reboot, then re-run installer if necessary.
Network failure — Network connectivity or download error while fetching payloads. 6 Action: Verify network connectivity, proxy, and certificates. Retry installation.
Package rejected during installation — Package rejected due to device security policy (MDM/Policy). 7 Action: Review device policy, allowlist the package or contact device admin.
Installation successful — Installation completed successfully. 8 Action: None required.

Add more rows for any custom or edge-case return codes your installer uses. For each code include: meaning, likely causes, and exact remediation steps (including log paths).
